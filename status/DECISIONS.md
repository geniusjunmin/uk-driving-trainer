## 决策记录

### [DEC-001] - 技术栈选型决策
* **日期**: 2026-06-17
* **决策 Agent**: Tech Lead / Architect Agent
* **决策详情**:
  * 采用 **Three.js** 作为 3D 渲染核心，因为 WebGL 适配良好，相比 React-Three-Fiber 更容易在底层定制渲染循环。
  * 采用 **TypeScript** 进行严格类型检查，避免大规模规则逻辑下的隐式类型 Bug。
  * 采用 **Vite** 作为工程打包脚手架，以获得秒级的开发热重载 (HMR) 速度。
  * 采用 **@dimforge/rapier3d-compat** 作为物理引擎，基于 WASM 编写，摩擦力、刚体受力和碰撞检测性能优越，适合车辆模拟。
* **背景/动因**:
  * 英国驾驶模拟器核心是高频率的碰撞边界、压线和路口距离判定，原生的 Three.js 与 3D 物理刚体配合，比 React 声明式框架更容易直接操作更新循环，方便后续规则引擎的精确介入。
* **影响范围**: 全局项目脚手架与物理代码。
* **状态**: 🟢 已批准 (Approved)
